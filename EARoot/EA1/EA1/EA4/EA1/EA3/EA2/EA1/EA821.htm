El inventario de datos de la organización se captura como un catálogo dentro del repositorio de arquitectura. Los catálogos son de naturaleza jerárquica y capturan una descomposición de una entidad metamodelo y también descomposiciones entre entidades de modelo relacionadas (por ejemplo, componente de datos lógicos -#gt; componente de datos físicos-#gt; entidad de datos).<br/><br/>Los catálogos forman la materia prima para el desarrollo de matrices y diagramas y también actúan como un recurso clave para administrar el negocio y la capacidad de TI.<br/><br/>Durante la fase de Arquitectura del negocio, se creó un diagrama de Servicio/Información del Negocio que muestra las entidades de datos clave requeridas por los principales servicios del negocio. Este es un requisito previo para el éxito de las actividades de arquitectura de datos.<br/><br/>Utilizando la trazabilidad desde la aplicación hasta la función del negocio y la entidad de datos inherente al marco de contenido, es posible crear un inventario de los datos necesarios para respaldar la Visión de la Arquitectura.<br/><br/>Una vez que los requisitos de datos se consolidan en una sola ubicación, es posible refinar el inventario de datos para lograr la coherencia semántica y eliminar lagunas y superposiciones.<br/><br/>Los siguientes catálogos deben considerarse para el desarrollo dentro de una arquitectura de datos:<br/><br/><ul>
<li>Catálogo de entidades de datos/componentes de datos</li></ul>
<br/>La estructura de los catálogos se basa en los atributos de las entidades metamodelo, tal como se definen en la <i>Parte IV, 30. Metamodelo de contenido .</i><br/>
