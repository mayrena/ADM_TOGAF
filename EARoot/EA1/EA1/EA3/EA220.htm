<br/>La Especificación de requisitos de arquitectura proporciona un conjunto de declaraciones cuantitativas que describen lo que un proyecto de implementación debe hacer para cumplir con la arquitectura. Una Especificación de Requisitos de Arquitectura normalmente formará un componente importante de un contrato de implementación o contrato para una Definición de Arquitectura más detallada.<br/><br/>La Especificación de Requisitos de Arquitectura actualizada, es un complemento del Documento de Definición de Arquitectura, con un objetivo complementario:<br/><br/><ul>
<li>El documento de definición de arquitectura proporciona una visión cualitativa de la solución y tiene como objetivo comunicar la intención del arquitecto.</li></ul>
<ul>
<li>La especificación de requisitos de arquitectura proporciona una visión cuantitativa de la solución, indicando los criterios medibles que deben cumplirse durante la implementación de la arquitectura.</li></ul>
<br/>Los contenidos típicos de una especificación de requisitos de arquitectura son:<br/><br/><ul>
<li>Medidas de éxito</li><li>Requisitos de arquitectura</li><li>Contratos de servicios de negocios</li><li>Contratos de servicios de aplicaciones</li><li>Directrices de aplicación</li><li>Especificaciones de implementación</li><li>Normas de aplicación</li><li>Requisitos de interoperabilidad</li><li>Requisitos de gestión de servicios de TI</li><li>Restricciones</li></ul>

<br/>Recurso obtenido de la <b><i>Fase Gestión de Requisitos (Salida):</i></b><br/><br/>Los resultados del proceso de gestión de requisitos pueden incluir, entre otros:<br/><ul>
<li>Especificación de requisitos de arquitectura actualizada (<i>Parte IV, 32.2.6 Especificación de requisitos de arquitectura</i>), si es necesario</li></ul>
<br/>El Repositorio de requisitos de arquitectura se actualizará como parte de la fase de gestión de requisitos y debe contener toda la información de requisitos.<br/><br/>Cuando surgen nuevos requisitos, o se cambian los existentes, se genera una Declaración de impacto de requisitos, que identifica las fases del ADM que deben revisarse para abordar los cambios. La declaración pasa por varias iteraciones hasta la versión final, que incluye todas las implicaciones de los requisitos (por ejemplo, costos, escalas de tiempo y métricas comerciales) en el desarrollo de la arquitectura. Una vez que se hayan finalizado los requisitos para el ciclo actual de ADM, se debe actualizar la Especificación de requisitos de arquitectura.<br/><br/>-Este entregable es un documento ya definido durante alguna fase previa del ADM y es consumido y producido (puede tener alguna modificación) durante la fase actual.
